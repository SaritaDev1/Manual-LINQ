# Capa de presentación

La capa de presentación es la parte del sistema que ve y utiliza el usuario.

En esta capa se encuentran las pantallas, formularios, botones, cajas de texto y tablas donde se ingresa o muestra información.

Esta capa no debe conectarse directamente a la base de datos ni escribir consultas LINQ directamente. Su trabajo es recibir los datos que ingresa el usuario y enviarlos a la capa de negocio.

Tomando como ejemplo la base de datos `Northwind`, si el usuario desea buscar clientes por país, la capa de presentación recibe el país escrito y llama a un método de la capa de negocio.

```csharp
private void btnBuscar_Click(object sender, EventArgs e)
{
    string pais = txtPais.Text;

    var clientes = negocioCliente.ObtenerClientesPorPais(pais);

    dgvClientes.DataSource = clientes;
}
```

En este ejemplo, `txtPais` representa una caja de texto donde el usuario escribe el país que desea buscar.

La variable `pais` guarda el valor escrito por el usuario.

Luego, `negocioCliente.ObtenerClientesPorPais(pais)` llama a la capa de negocio y le envía ese dato para que pueda procesarlo.

El resultado se guarda en la variable `clientes`. Esa variable puede contener una lista de objetos de la entidad `Customer`.

Finalmente, `dgvClientes.DataSource = clientes` muestra la información en una tabla o grilla de la pantalla.

En esta capa no se escribe directamente la consulta LINQ. LINQ participa después, en la capa de negocio o en la capa de acceso a datos, donde se filtra o consulta la información de `Northwind`.

La entidad `Customer` ayuda a transportar los datos del cliente entre las capas. Por ejemplo, cada cliente que se muestra en la tabla puede venir representado como un objeto `Customer`.

La capa de presentación sirve para interactuar con el usuario, enviar datos a la capa de negocio y mostrar los resultados, pero no debe consultar directamente la base de datos.
